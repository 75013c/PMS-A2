Privacy and Security Analysis

This page explains privacy and security considerations of the Inventory Management System.
It explains how the application protects data accuracy, reduces avoidable risks and helps users understand important system limitations.

Key Privacy and Security Requirements

Reduce incorrect changes

The system could minimise incorrect user input and prevent accidental deletion through validation and confirmation mechanisms.

Limit privacy exposure

The application avoids collecting unnecessary personal information and only store data that is required for inventory management purposes.

Security Considerations

Input Validation

The system validates user input before saving data. Required fields must be completed, and numerical values such as quantity and price must be valid and non-negative.

Unique Item Identification

The application checks that each item ID is unique. This helps protect data integrity and prevents duplicate inventory records from being created accidentally.

Controlled Record Changes

Update and delete actions are separated into clear workflows. The delete process also includes a confirmation step, which helps reduce accidental data loss.
